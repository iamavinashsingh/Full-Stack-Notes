# Lecture 05: CSS Grid (Quick Notes)

## 1. Why Grid?
- **Flexbox = 1D** (row OR column).
- **Grid = 2D** (rows + columns together).
- Old way = nested flexboxes â†’ messy HTML.
- Grid â†’ clean HTML + CSS controls layout.

---

## 2. Grid Vocabulary
- **Grid Lines** â†’ numbered fences (start from 1).
- **Track** â†’ space between 2 lines (row/column).
- **Cell** â†’ smallest unit (row + col intersection).
- **Area** â†’ rectangle made of multiple cells.

---

## 3. Defining Grid (Container Blueprint)
- `display: grid;`
- **Columns** â†’ `grid-template-columns`
  - `200px 200px 200px` (fixed)
  - `25% 50% 25%` (percent)
  - `100px auto 100px` (mixed)
  - `1fr 1fr 1fr` (equal shares)
  - `2fr 1fr` (ratio)
- **Rows** â†’ `grid-template-rows`
- **repeat()** â†’ `repeat(12, 1fr)`
- **gap** â†’ space between tracks (`gap: 20px;`)

---

## 4. Placing Items
- Default = auto-placement (Lâ†’R, Tâ†’B).
- Manual placement:
  - `grid-column: 1 / 3;`
  - `grid-row: 2 / 4;`
- Shorthands:
  - `grid-column: span 2;` â†’ span 2 cols
  - `grid-row: span 3;` â†’ span 3 rows

---

## 5. Example: Holy Grail Layout
HTML:
```html
<body class="grid-container">
  <header>Header</header>
  <nav>Navigation</nav>
  <main>Main</main>
  <aside>Sidebar</aside>
  <footer>Footer</footer>
</body>
```
```css
.grid-container {
  display: grid;
  grid-template-columns: 1fr 3fr 1fr; /* sidebar-main-sidebar */
  grid-template-rows: auto 1fr auto; /* header-content-footer */
  gap: 20px;
}
header { grid-column: 1 / 4; }
footer { grid-column: 1 / 4; }
```

# Lecture 06: Media Query, Shadows & Overflow

## 1. Media Queries
- Purpose â†’ Make sites adapt to screen sizes (responsive design).
- Syntax â†’ `@media media-type and (condition) { ... }`
- Common features:
  - `max-width`: applies up to given size (desktop-first).
  - `min-width`: applies from given size upwards (mobile-first).
  - Combine â†’ `@media screen and (min-width:768px) and (max-width:1023px)`

### Example
```css
/* Mobile first */
.container { grid-template-columns: 1fr; }

/* Tablet and up */
@media (min-width:768px) {
  .container { grid-template-columns: 1fr 1fr; }
}

/* Desktop and up */
@media (min-width:1024px) {
  .container { grid-template-columns: 1fr 1fr 1fr; }
}
```
## 2. Box Shadows

**Property:** `box-shadow: offsetX offsetY blur spread color;`

- **offsetX/Y** â†’ shadow position
- **blur** â†’ softness (0 = sharp, >0 = soft)
- **spread** â†’ bigger (+) / smaller (-) than box
- **color** â†’ use `rgba()` for subtle effect
- **inset** â†’ inner shadow
- **Multiple shadows** â†’ separate with `,`

```css
.card {
  box-shadow: 0 4px 15px rgba(0,0,0,0.1);
}
```
## 3. Overflow

Controls content bigger than the box.

### Values:
- **visible** â†’ content spills out (default).
- **hidden** â†’ extra content cut off.
- **scroll** â†’ always show scrollbars.
- **auto** â†’ scrollbars only when needed (best choice).

Can target separately â†’ `overflow-x`, `overflow-y`.

### Example:
```css
.box {
  height: 100px;
  overflow: auto;
}
```

- **Media Query** â†’ responsive layouts.
- **Box Shadow** â†’ depth & realism.
- **Overflow** â†’ control extra content.

# Lecture 07: CSS Animation (Quick Notes)

## 1. What is Animation?
- **Definition** â†’ Change in style over time.  
- **Analogy** â†’ Sunset (blue sky â†’ orange sky).  
- Normal CSS = only 1 state. Animation = full journey.  

---

## 2. Two-Part System
1. **Storyboard (@keyframes)** â†’ define start & end states.  
2. **Director (animation property)** â†’ tell element how to play it.  

---

## 3. @keyframes Syntax
```css
@keyframes name {
  from { /* start */ }
  to   { /* end */ }
}
```
```css
@keyframes sunset {
  from { background: skyblue; }
  to   { background: orangered; }
}

```
---
## 4. Applying Animation
```css
.sky {
  animation-name: sunset;
  animation-duration: 5s;
}
```

Plays once â†’ then stops.

 ## 5. Repetition Controls

- animation-iteration-count: infinite; â†’ loop forever.

- animation-direction: alternate; â†’ play forward then backward (smooth).

## 6. Keep Final State

- Problem â†’ element snaps back after animation ends.

- Solution â†’ animation-fill-mode: forwards;

## 7. Example: Progress Bar
```css
@keyframes fill-bar {
  from { width: 0%; }
  to   { width: 100%; }
}

.progress-fill {
  animation: fill-bar 4s forwards;
}

```
âœ… Bar fills from 0% â†’ 100% and stays filled.

## ðŸ”‘ Key Takeaways

- Animation = style + time.

- @keyframes = what happens.

- animation = how it happens.

- Control â†’ duration, loop, direction, fill-mode.

# Lecture 08: CSS Transform & Transition (Quick Notes)

## 1. Transform (Change element without breaking layout)
- Works after layout â†’ smooth, independent movement.  
- Core Functions:
  - **translate(x,y)** â†’ move â†’ `translateX(50px)`  
  - **scale()** â†’ resize â†’ `scale(1.2)` (bigger), `scale(0.9)` (smaller)  
  - **rotate(deg)** â†’ turn â†’ `rotate(45deg)`  
  - **skew()** â†’ slant â†’ `skewX(15deg)`  
- Combine â†’ `transform: translateX(50px) rotate(10deg) scale(1.2);`  

---

## 2. Transition (Make style changes smooth)
- Normal `:hover` â†’ instant snap.  
- **transition** makes it gradual.  
- Syntax:  
  `transition: property duration timing delay;`  

### Parts
1. **property** â†’ what to animate (`all`, `transform`, `background-color`)  
2. **duration** â†’ how long (`0.3s`)  
3. **timing-function** â†’ pacing (`ease`, `linear`, `ease-in-out`)  
4. **delay** â†’ when to start  

---

## 3. Practical Example (Lifting Card)
```css
.card {
  transition: transform 0.3s ease-in-out, box-shadow 0.3s ease-in-out;
}
.card:hover {
  transform: translateY(-10px);
  box-shadow: 0 10px 20px rgba(0,0,0,0.2);
}
```
âœ… On hover â†’ card smoothly lifts & shadow grows.

---

## ðŸ”‘ Takeaways

- Transform = What changes (move/resize/rotate/skew).

- Transition = How changes feel (smooth timing).

- Always put transition on the base state, not :hover.
