# CSS Notes

## What is CSS?

**CSS** = Cascading Style Sheets  
It makes websites look **pretty** (colors, fonts, layouts)

- **Main Idea**: HTML handles content, **CSS handles appearance**

---

## Why We Need CSS

Without CSS:

- Websites looked plain and boring  
- Had to style each element individually  
- Code was messy and hard to maintain  

---

## How to Add CSS

### 1. Inline CSS (For one element)

```html
<h1 style="color: blue;">Blue Heading</h1>
```

### 2. Internal CSS (For one page)

```html
<style>
  h1 { color: blue; }
</style>
```

### 3. External CSS (Best for whole website)

```html
<link rel="stylesheet" href="styles.css">
```

---

## CSS Building Blocks

```css
selector {
  property: value;
}
```

- **Selector**: Who to style (like `h1`)
- **Property**: What to change (like `color`)
- **Value**: How to change it (like `blue`)

---

## 4 Main Selectors

### 1. Element Selector

Styles all elements of that type

```css
p {
  color: gray;
}
```

### 2. Class Selector (Most common)

Styles elements with that class

```css
.warning {
  color: red;
}
```

```html
<p class="warning">This is red</p>
```

### 3. ID Selector

Styles one specific element

```css
#header {
  background: blue;
}
```

```html
<header id="header">...</header>
```

### 4. Group Selector

Styles multiple elements at once

```css
h1, h2, h3 {
  font-family: Arial;
}
```

---

## Colors in CSS

### 1. Color Names (Easiest)

```css
color: red;
color: lightblue;
```

### 2. Hex Codes (Most common)

```css
color: #FF0000;    /* Red */
color: #0000FF;    /* Blue */
color: #000000;    /* Black */
color: #FFFFFF;    /* White */
```

### 3. RGB (Red, Green, Blue)

```css
color: rgb(255, 0, 0);        /* Red */
color: rgba(255, 0, 0, 0.5);  /* See-through red */
```

### 4. HSL (Hue, Saturation, Lightness)

```css
color: hsl(0, 100%, 50%);       /* Red */
color: hsla(0, 100%, 50%, 0.5); /* See-through red */
```

# CSS Basics: Box Model, Display, Cascade & Position

## Everything is a Box
Your browser sees every element on a webpage as a rectangular box - even circles are just boxes with rounded corners!

### The 4 Layers of a Box
Every box has 4 layers (from inside out):
1. **Content** - Your text or image
2. **Padding** - Space inside the box (like the mat in a picture frame)
3. **Border** - The edge of the box (like the picture frame itself)
4. **Margin** - Space outside the box (personal space between frames)

### Controlling Box Size
- `width` & `height`: Set content size
- `max-width`: "Grow up to this size but no more" (great for responsive design)

```css
.container {
  width: 100%;        /* Fill container */
  max-width: 800px;   /* But never wider than 800px */
}
```
# üß† CSS Quick Notes - Simple & Easy

## üì¶ Spacing: Padding & Margin

### üü© Padding (Inside Space)
```css
padding: 10px;                /* All sides */
padding: 10px 20px;           /* Top/Bottom = 10px, Left/Right = 20px */
padding: 10px 20px 30px 40px; /* Top, Right, Bottom, Left */
```

### üü• Margin (Outside Space)
Same shorthand as padding!

---

## üî≤ Borders

### 3 Parts:
- `border-width` ‚Üí Thickness  
- `border-style` ‚Üí solid, dotted, dashed, etc.  
- `border-color` ‚Üí Border color  

### ‚úÖ Shorthand:
```css
border: 2px solid black;
```

---

## üü° Rounded Corners

```css
border-radius: 10px;                  /* All corners */
border-radius: 10px 5px 10px 5px;     /* TL, TR, BR, BL */
```

---

## üßô‚Äç‚ôÇÔ∏è The Box-Sizing Miracle

### Problem:  
Setting `width: 300px` adds **padding + border** on top.

### ‚úÖ Fix:
```css
* {
  box-sizing: border-box;
}
```
> Now `width` includes padding and border. Much easier to manage!

---

## üß± Block vs. Inline Elements

### üì¶ Block Elements
- Start on a new line  
- Full width  
- Stack vertically  

**Examples:** `<div>`, `<p>`, `<h1>`, `<section>`

### üßµ Inline Elements
- Stay on same line  
- Only take as much space as needed  
- Flow horizontally  

**Examples:** `<span>`, `<a>`, `<strong>`

---

## üõ†Ô∏è Changing Display Type

```css
a { display: block; }          /* Inline ‚Üí Block */
li { display: inline; }        /* Block ‚Üí Inline */
button { display: inline-block; } /* Inline but supports width/height */
.hidden { display: none; }     /* Hide element */
```

---

## üîÑ Block vs Inline - Comparison

| Feature              | Block ‚úÖ | Inline ‚ùå |
|----------------------|----------|-----------|
| Starts new line      | ‚úÖ Yes   | ‚ùå No     |
| Width                | Full     | Auto      |
| Height               | Respects | Ignores   |
| Margin (top/bottom)  | ‚úÖ Works | ‚ùå Ignored|
| Margin (left/right)  | ‚úÖ Works | ‚úÖ Works  |
| Padding (top/bottom) | ‚úÖ Works | ‚úÖ Visual only |
| Padding (left/right) | ‚úÖ Works | ‚úÖ Works  |

---

## üß¨ CSS Cascade - Which Rule Wins?

1. `!important` ‚Üí üö® Emergency override  
   ```css
   #text { color: green !important; }
   ```

2. **Inline styles** ‚Üí Closest  
   ```html
   <p style="color: red;">Text</p>
   ```

3. **ID Selector** ‚Üí Unique  
   ```css
   #sidebar { background: gray; }
   ```

4. **Class Selector** ‚Üí Reusable group  
   ```css
   .warning { color: orange; }
   ```

5. **Element Selector** ‚Üí General rule  
   ```css
   p { color: black; }
   ```

6. **Source Order** ‚Üí Last defined wins if all else is equal

---

## üìç Position Property

### 1. `static` (default)
- Normal flow  
- Ignores top/right/bottom/left

### 2. `relative` (nudge)
- Stays in place, but you can move it  
- Used to position **absolute children**

```css
.relative-box {
  position: relative;
  top: 20px;
  left: 20px;
}
```

### 3. `absolute` (free position)
- Removed from normal flow  
- Positioned relative to nearest **positioned** ancestor  

```css
.parent { position: relative; }
.child  { position: absolute; top: 10px; right: 10px; }
```

### 4. `fixed` (stuck to screen)
- Relative to viewport  
- Stays in place while scrolling

```css
.nav {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
}
```

### 5. `sticky` (hybrid)
- Starts as relative  
- Becomes fixed after scroll threshold

```css
.header {
  position: sticky;
  top: 0;
}
```

---

## üóùÔ∏è Key Takeaways

- Everything is a box in CSS  
- Use `box-sizing: border-box` for sanity  
- `Padding` = inside space  
- `Margin` = outside space  
- Block elements stack, inline elements flow  
- Use `display` to change layout behavior  
- CSS Cascade:  
  `!important > Inline > ID > Class > Element > Source Order`  
- Know your `position`:  
  `static`, `relative`, `absolute`, `fixed`, `sticky`

# üéØ CSS Flexbox - Quick & Easy Notes

---

## üìå Why Flexbox?

**Old Problems (Before Flexbox):**
- Vertical centering = complex hacks  
- Equal-height columns = hard (needed JS or backgrounds)  
- Spacing items evenly = messy margin math  
- Used `float` (not meant for layout) ‚Üí led to bugs + clearfix hacks

**Flexbox = Solution**  
A layout model made for arranging items in **one direction** (row or column) with built-in alignment, spacing, and reordering tools.

---

## üìê Vertical Centering Example

### ‚ùå Old Way (Hacky)
```html
<div class="parent-box">
  <div class="child-box">Center Me!</div>
</div>
```
```css
.parent-box { position: relative; height: 300px; }
.child-box {
  position: absolute;
  top: 50%; left: 50%;
  transform: translate(-50%, -50%);
}
```

### ‚úÖ Flexbox Way (Easy)
```css
.parent-box {
  display: flex;
  justify-content: center;  /* horizontal */
  align-items: center;      /* vertical */
  height: 300px;
}
```

---

## üë• Flex Container & Items

### Two Key Roles:
- **Flex Container** ‚Üí Parent with `display: flex;`
- **Flex Items** ‚Üí All *direct children* of the container

> üî∏ *Only direct children become flex items.* Grandchildren are not affected unless their parent is also a flex container.

---

## üß≠ Flex Container Properties

### 1. `flex-direction`: Main Axis
```css
/* Values: */
row              /* Default: left to right */
column           /* Top to bottom */
row-reverse
column-reverse
```

### 2. `justify-content`: Align on Main Axis
```css
flex-start       /* Items start at beginning */
flex-end         /* Items at the end */
center           /* Items in the center */
space-between    /* Equal space between */
space-around     /* Space around items */
space-evenly     /* Equal space everywhere */
```

### 3. `align-items`: Align on Cross Axis
```css
stretch          /* Default: stretch to fill */
flex-start       /* Align to top (row) or left (column) */
flex-end         /* Align to bottom or right */
center           /* Perfect vertical or horizontal centering */
```

### 4. `flex-wrap`: Handle Overflow
```css
nowrap           /* Default: no wrapping */
wrap             /* Items wrap to next line */
```

### 5. `gap`: Space Between Items
```css
gap: 20px;             /* Row and column gap */
row-gap: 10px;
column-gap: 30px;
```

---

## üß± Common Layout Questions Solved by Flexbox

| Question                          | Property         |
|----------------------------------|------------------|
| Row or column?                   | `flex-direction` |
| Space between items?             | `justify-content`|
| Vertical alignment?              | `align-items`    |
| Handle overflow/wrapping?        | `flex-wrap`      |
| Add spacing between items only? | `gap`            |

---

## üîß Flex Item Properties

### 1. `flex-grow`: Take Extra Space
```css
flex-grow: 1;   /* Item will grow to fill space */
```
> If one item has `flex-grow: 1` and others have 0, it takes all extra space.

### 2. `flex-shrink`: Shrink When Needed
```css
flex-shrink: 0;   /* Don't shrink (default = 1) */
```

### 3. `flex-basis`: Preferred Size
```css
flex-basis: 200px;   /* Like width, but flexible */
```

### 4. `flex`: Shorthand
```css
flex: 1;          /* grow: 1, shrink: 1, basis: 0% */
flex: auto;       /* grow: 1, shrink: 1, basis: auto */
flex: none;       /* grow: 0, shrink: 0 */
```

### 5. `align-self`: Override `align-items`
```css
align-self: flex-start;   /* For individual item alignment */
```

---

## üß† Key Flexbox Concepts Recap

- Flexbox is for **1D layout**: row or column  
- Use `display: flex` on the **container**
- Direct children become **flex items**
- Flexbox solves:
  - Vertical centering
  - Equal heights
  - Spacing and alignment
  - Responsive wrapping
- Use `gap` instead of margins
- Flex items can **grow, shrink, and align independently**

---

üìö **Cheat Sheet Summary**
| Property         | Applies To | Purpose                         |
|------------------|------------|---------------------------------|
| `display: flex`  | Container  | Start using Flexbox             |
| `flex-direction` | Container  | Row or column layout            |
| `justify-content`| Container  | Align items on main axis        |
| `align-items`    | Container  | Align items on cross axis       |
| `flex-wrap`      | Container  | Wrap items to next line         |
| `gap`            | Container  | Space between items             |
| `flex-grow`      | Item       | Grow to fill space              |
| `flex-shrink`    | Item       | Shrink when needed              |
| `flex-basis`     | Item       | Default size before grow/shrink|
| `flex`           | Item       | Shorthand for grow/shrink/basis|
| `align-self`     | Item       | Override container alignment    |

---
